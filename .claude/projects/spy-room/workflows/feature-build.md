# Feature Build Workflow

1. Read `project-state.md` for current priorities
2. Read `architecture.md` to understand where the feature fits
3. Read existing screen/component code that the feature touches
4. Check `ui-ux.md` for styling rules
5. Implement with minimal changes — follow existing patterns
6. Verify by re-reading modified files
7. Update `project-state.md` and `session-log.md`
